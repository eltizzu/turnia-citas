# Documentos Vivos De Turnia

Este archivo define que documentos deben mantenerse actualizados durante el desarrollo.

## Regla de mantenimiento

Cada avance importante debe revisar si impacta en alguno de estos documentos:

- `DOCUMENTO_COMERCIAL.md`
- `MANUAL_DE_USO.md`
- `CHECKLIST_FINAL_PRODUCTO.md`
- `RESUMEN_LEGAL_Y_FISCAL_INICIAL.md`
- `WEB_LEGAL_CONTENIDOS.md`
- `ECOSISTEMA_IDEAS_FUTURAS.md`

Si cambia una funcion, flujo, perfil, decision comercial, decision legal, alcance o idea futura, actualizar el documento correspondiente en el mismo bloque de trabajo.

## Cuando actualizar cada uno

### Documento Comercial

Actualizar si cambia:

- publico objetivo;
- propuesta de valor;
- modulos;
- precio;
- discurso de venta;
- diferenciacion;
- estado del producto.

### Manual De Uso

Actualizar si cambia:

- pantalla;
- boton;
- flujo;
- estado de cita;
- perfil de usuario;
- accion importante.

### Checklist Final Del Producto

Actualizar si cambia:

- estado de QA;
- pantalla lista/no lista;
- pendiente importante;
- preparacion para demo;
- preparacion para piloto.

### Resumen Legal Y Fiscal Inicial

Actualizar si cambia:

- datos tratados;
- pais objetivo;
- forma de cobro;
- integraciones;
- roles frente a datos;
- obligaciones detectadas.

### Web Legal

Actualizar si cambia:

- web publica;
- formularios;
- textos legales;
- cookies;
- analitica;
- titular legal;
- marca/producto.

### Ecosistema / Ideas Futuras

Actualizar si aparece:

- idea nueva;
- modulo futuro;
- producto hermano;
- decision de sacar algo del MVP;
- decision de meter algo al MVP.

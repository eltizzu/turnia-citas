# Contexto Inicial De Turnia

## 1. Que Es Este Proyecto

Turnia es un nuevo producto separado de BrasaFlow.

La idea es crear una app de citas para rubros que trabajan por agenda y turnos, como:

- Peluquerias
- Barberias
- Unas
- Estetica
- Masajes
- Centros de belleza
- Servicios similares

No es un proyecto de hosteleria.

## 2. Por Que Nace

La decision fue empezar este producto porque:

- Es mas simple de cerrar que otros productos hermanos.
- Se puede vender mas facil y mas rapido.
- Tiene mucho sentido para ofrecerlo desde Impulso Web.
- Encaja muy bien con futuras integraciones como WhatsApp, recordatorios y confirmaciones.

## 3. Enfoque Del Producto

La app tiene que ser:

- Muy clara
- Muy visual
- Muy facil de usar
- Pensada primero para celular
- Comoda para gente joven y para gente mas grande

No tiene que sentirse como un panel frio. Tiene que sentirse como una agenda moderna, liviana y facil.

## 4. Lo Que Ya Se Decidio

### Direccion General

- Primero se construye la base del producto.
- El nombre Turnia queda como nombre de trabajo.
- El foco inicial esta en la experiencia real del negocio, no en meter muchas funciones de golpe.
- La agenda es la pantalla principal.

### Alcance Inicial

La version base deberia cubrir:

- Profesionales
- Servicios
- Agenda del dia
- Agenda semanal
- Creacion de citas
- Edicion de citas
- Cancelacion
- Reprogramacion
- Estados de citas
- Clientes
- Notas basicas
- Enlace publico de reserva

### Funciones Futuras Importantes

- Recordatorios automaticos
- Confirmacion de citas
- Reprogramacion desde mensaje
- Integracion con WhatsApp
- IA para responder consultas frecuentes
- Estadisticas de ocupacion
- Bloqueo de horarios
- Multi-sede

## 5. Relacion Con Otros Productos

Este proyecto no tiene que mezclarse con BrasaFlow.

BrasaFlow queda como producto de gestion interna para hosteleria. Turnia queda como sistema de agenda y citas para otros rubros.

## 6. Estrategia

Turnia no deberia competir como software enorme de gestion. Deberia competir como agenda simple, movil, cercana y sin comisiones para salones pequenos y medianos.

Principio del producto:

Primero simple. Despues automatico. Nunca pesado.

## 7. Siguiente Paso

Pasar de la demo visual a una app funcional con datos reales de prueba:

- Agenda
- Profesionales
- Servicios
- Clientes
- Nueva cita
- Estados
- Primer flujo de confirmacion por WhatsApp

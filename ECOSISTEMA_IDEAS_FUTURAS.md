# Documento De Ecosistema / Ideas Futuras

Estado: documento vivo. Actualizar cuando aparezcan nuevas ideas, modulos, productos hermanos o cambios de alcance.

## Producto principal actual

Turnia es, por ahora, una app de agenda y reservas para negocios de servicios por turno.

El producto principal incluye:

- agenda;
- link publico de reserva;
- servicios;
- profesionales;
- clientes;
- bloqueos;
- reportes;
- exportacion;
- configuracion;
- WhatsApp preparado;
- login y base real como siguiente hito.
- identidad visual inicial con logo y favicon.

Todo lo que no ayude directamente a vender o validar esto queda como futuro.

## Principio de enfoque

No mezclar ideas verdes con el MVP.

Regla:

- si ayuda a que un salon reserve y gestione mejor, puede entrar;
- si abre otro negocio distinto, va a ideas futuras;
- si requiere mucha infraestructura antes de validar, se pospone.

## Actualizaciones futuras dentro de Turnia

### Automatizacion de mensajes

- recordatorios automaticos;
- confirmacion automatica avanzada;
- mensajes de reprogramacion;
- mensajes post-cita;
- solicitud de resenas.

Prioridad: media. Primero validar WhatsApp semiautomatico.

### Lista de espera

- cliente pide horario no disponible;
- negocio ve interesados;
- Turnia avisa cuando se libera hueco.

Prioridad: media-alta para salones con mucha demanda.

### Reactivacion de clientes

- clientes que no vuelven hace X semanas;
- mensaje sugerido;
- campanas simples.

Prioridad: futura.

### Ocupacion y analitica avanzada

- ocupacion por profesional;
- horas muertas;
- comparativa mensual;
- no-show por cliente;
- servicio mas rentable.

Prioridad: despues de tener datos reales.

### Pagos online

- senas;
- pago completo;
- politica de cancelacion;
- integracion Stripe/MercadoPago segun pais.

Prioridad: futura. Evitar al principio por complejidad legal/fiscal/operativa.

### Multi-sucursal

- varias sedes;
- profesionales por sede;
- reportes por sede.

Prioridad: baja para MVP.

### Roles avanzados

- admin;
- recepcion;
- profesional;
- solo lectura;
- permisos por accion.

Prioridad: media despues de pilotos.

## Productos hermanos posibles

Estos no pertenecen al MVP actual.

### Turnia Pro

Version para negocios mas grandes:

- multi-sucursal;
- permisos avanzados;
- analitica;
- integraciones;
- soporte premium.

### Turnia Solo

Version ultra simple para profesionales independientes:

- una agenda;
- un link;
- pocos reportes;
- precio bajo.

### Turnia CRM

Modulo centrado en clientes:

- historial avanzado;
- segmentos;
- cumpleanos;
- reactivacion;
- campanas.

### Turnia Payments

Modulo de cobros:

- senas;
- pagos online;
- cancelaciones;
- conciliacion.

### Turnia Reviews

Modulo de resenas:

- pedir resena tras cita;
- seguimiento;
- plantilla de mensaje;
- panel de reputacion.

## Criterios para sumar algo al producto principal

Antes de sumar una funcion, preguntar:

1. Lo pedirian al menos 2 o 3 negocios reales?
2. Reduce trabajo manual?
3. Ayuda a conseguir o retener clientes?
4. Es facil de explicar?
5. Se puede vender?
6. No complica demasiado el soporte?
7. No aumenta mucho el riesgo legal/fiscal?

Si la respuesta es dudosa, dejarlo en este documento y no meterlo todavia.
